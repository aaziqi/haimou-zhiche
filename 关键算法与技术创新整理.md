# 海眸智澈：关键算法、关键技术与技术创新整理

> 本文档用于填写“关键算法、关键技术、技术创新”等内容，可直接摘取到作品说明书、设计文档、申报书或答辩材料中。

## 一、总体概述

本项目面向水下图像在真实场景中普遍存在的偏色、低对比、散射模糊和细节丢失等问题，构建了一个集**图像增强、方法对比、指标评测、下游检测验证、案例归档和本地交互展示**于一体的完整软件系统。项目不仅关注增强算法本身，还关注算法如何以系统化方式落地，形成“可演示、可验证、可复现、可答辩”的作品形态。

从算法层面，系统以 CycleGAN 无配对图像转换框架为基础，构建了改进模型 MP-CycleGAN；从工程层面，系统实现了传统方法、基线模型、改进模型的统一接入，并将指标计算、可视化展示、报告导出和归档管理整合到统一平台中。

## 二、关键算法

### 1. 基于 CycleGAN 的无配对水下图像增强

本项目的核心增强框架建立在 CycleGAN 之上。CycleGAN 通过对抗学习和循环一致性约束，在没有严格配对样本的情况下学习“水下退化图像域”到“清晰图像域”的映射，因此非常适合真实水下场景中配对数据难以获取的问题。

其基础思想包括：

- 使用生成器将水下图像映射到增强域；
- 使用判别器约束生成结果逼近目标域分布；
- 使用循环一致性损失约束映射前后的内容尽量保持一致；
- 在无监督或弱监督条件下完成图像增强任务。

这一框架解决了“水下图像增强高度依赖配对数据”的现实难题，为后续改进模型提供了基础。

### 2. 改进模型 MP-CycleGAN

在标准 CycleGAN 的基础上，项目提出了面向水下场景的改进模型 MP-CycleGAN。其核心思想是：在对抗损失与循环一致性损失之外，引入符合水下视觉特点的多重约束，从而提升增强结果的自然性、结构保持能力和跨域稳定性。

主要改进包括以下几个方面：

#### （1）灰世界颜色约束

水下图像通常存在明显的蓝绿偏色。为缓解这种问题，项目引入基于灰世界假设的颜色约束，使增强结果在 RGB 通道统计上更加平衡，从而降低不自然的色偏。

作用：

- 抑制蓝绿偏色；
- 提升整体色彩自然度；
- 增强颜色恢复的稳定性。

#### （2）结构一致性约束

仅依赖 GAN 风格迁移，容易在增强过程中出现结构漂移、边缘模糊或纹理失真。为此，项目加入了基于梯度信息的结构一致性约束，保证增强前后的主要边缘和几何轮廓尽量保持一致。

作用：

- 保留目标轮廓与边缘信息；
- 减少细节错位和结构形变；
- 提高增强图像在下游任务中的可用性。

#### （3）感知损失约束

为了让增强结果不仅“像素上变化”，更在高层语义和纹理细节上保持合理，项目引入基于预训练 VGG 特征的感知损失。该约束在特征空间而非单纯像素空间对结果进行优化，能够更好保留图像的纹理与视觉层次。

作用：

- 保持语义一致性；
- 提高纹理清晰度；
- 改善主观观感。

#### （4）颜色统计一致性约束

为进一步控制增强图像的全局颜色分布，项目还设计了颜色统计一致性约束，用于约束通道均值与方差，使输出颜色更加稳定。

作用：

- 降低颜色过饱和或失真风险；
- 提升结果整体协调性；
- 为消融实验提供可解释支撑。

### 3. 传统增强基线算法

为了让评委和用户更清楚地理解“改进模型相较传统方法的优势”，系统同时接入了多种传统增强算法作为基线：

- Gray World 白平衡
- CLAHE 局部对比度增强
- Gray World + CLAHE 组合方法
- Gamma 校正

这些方法具有实现简单、速度快、可解释性强的特点，适合作为对照方案。但在复杂水下场景中，它们通常难以同时兼顾颜色恢复、结构保持与视觉自然度，因此也反衬出深度模型方案的必要性。

### 4. 多指标联合评测算法

项目采用“无参考指标 + 有参考指标”的双路径评测策略，以避免只靠主观观感或单一指标下结论。

主要指标包括：

- UCIQE：用于评价水下图像颜色与清晰度质量；
- UIQM：用于衡量水下图像主观视觉质量；
- PSNR：用于衡量增强结果与参考图像之间的像素保真度；
- SSIM：用于衡量结构相似性。

这一设计使系统既能反映增强图像的视觉效果，又能体现算法在客观意义上的可验证性。

### 5. 下游检测联动验证

项目没有把“增强”停留在图像层面，而是进一步接入 YOLO 检测器，对原图与增强图进行下游任务对比。通过比较检测数量、检测可视化结果与 JSON 输出，验证增强对真实视觉任务的支持作用。

这一点说明：

- 增强结果不仅“更好看”；
- 还可能提升目标检测、识别等后续任务的稳定性；
- 体现出作品面向应用场景的完整价值链。

## 三、关键技术

### 1. 无配对图像增强技术

项目采用无配对图像转换技术解决真实水下场景中缺乏严格配对样本的问题，这是本项目算法层面的基础关键技术。相比传统监督学习，无配对方法更适合实际海洋应用环境。

### 2. 水下图像颜色恢复与结构保持协同优化技术

项目不是单纯提升亮度或对比度，而是通过颜色约束、结构约束和感知约束共同优化，使增强结果在“颜色自然、细节清晰、结构稳定”之间取得平衡。

### 3. 本地交互式推理服务技术

项目采用“本地 Web 前端 + Python 推理服务”的结构，前端负责交互展示，后端负责模型调用、指标计算和文件归档。这样既保留了深度学习推理能力，又使系统具备良好的展示效果和答辩可操作性。

### 4. 多方案统一接入技术

系统将传统方法、原始 CycleGAN、改进模型 MPCGAN 统一纳入同一平台，解决了研究代码通常“方法分散、流程割裂、结果难比对”的问题。通过统一接口和统一展示层，用户能够快速进行方法切换和横向比较。

### 5. 自动报告生成与归档技术

系统能够自动输出 HTML 报告、CSV 指标文件、JSON 检测结果和案例归档索引，使每次演示或实验结果都能够沉淀、复用和回溯。

这一技术带来的价值包括：

- 减少人工整理成本；
- 提高结果可追溯性；
- 增强答辩时的可信度与系统完整度。

### 6. 静态展示与动态推理双模式技术

项目支持“本地动态推理模式”和“静态展示模式”两种运行方式。本地模式适合答辩时进行真实增强推理，静态模式适合部署展示页或进行线上传播。该技术提高了作品的灵活性和可部署性。

### 7. 面向答辩场景的前端可视化技术

系统前端并非简单工具页面，而是专门围绕比赛展示需求设计，具备：

- 首页成果卡片与封面展示；
- 增强前后可视化对比；
- 多方案横向对比界面；
- 图表页与指标展示；
- 案例归档与筛选展示；
- 一键演示流程页。

这使项目具备明显的软件作品特征，而不只是算法脚本的集合。

## 四、技术创新点

### 创新点 1：从“研究型算法代码”升级为“完整软件系统”

本项目最大的创新不只是提出改进模型，而是将原本偏研究型的增强算法，重构为一个完整的软件应用系统。系统具备交互、展示、评测、归档和报告功能，使作品真正符合“软件应用与开发”类竞赛的作品要求。

可概括为：

- 算法可运行；
- 结果可展示；
- 过程可交互；
- 输出可沉淀；
- 价值可表达。

### 创新点 2：构建“传统方法—基线模型—改进模型”的完整技术演进链

项目并未只展示“最好结果”，而是系统性地引入传统算法、原始 CycleGAN 和改进模型，构建完整对比链条。这种设计既能帮助评审理解技术路线，又能突出改进方案的必要性和进步点。

这一创新的价值在于：

- 强化技术可解释性；
- 提升答辩表达逻辑；
- 增强实验结论的说服力。

### 创新点 3：多感知驱动的 MP-CycleGAN 设计

在算法层面，项目针对水下图像退化特性，将灰世界颜色约束、结构一致性约束和感知损失协同引入 CycleGAN，形成多感知驱动的无配对增强模型。这一设计兼顾了颜色恢复、结构保持与纹理表达，相比标准 CycleGAN 更适合水下图像增强任务。

### 创新点 4：把增强结果与下游检测任务联动验证

传统水下增强工作往往停留在图像主观效果和单纯指标层面，而本项目进一步将增强结果接入 YOLO 检测器，形成“增强—检测”的应用验证链条。这样能够从任务层面说明增强的工程价值，而不是仅停留在视觉层面。

### 创新点 5：构建可复现、可归档、可传播的成果组织方式

系统支持案例归档、报告导出、图表展示和静态数据回退，使作品结果能够被重复查看、反复使用和多场景传播。这种设计提升了系统的工程成熟度，也增强了作品在竞赛、教学和演示中的适应性。

### 创新点 6：兼顾算法深度与竞赛展示表达

很多项目要么算法强但展示弱，要么界面好看但技术支撑不足。本项目的创新在于把两者有机结合：

- 算法上有明确改进；
- 系统上有完整流程；
- 展示上有统一叙事；
- 结果上有数据支撑；
- 应用上有下游验证。

这使项目既能体现技术深度，又具备较强的现场说服力。

## 五、可直接用于文档填写的精简版

### 1. 关键算法（精简版）

本项目以 CycleGAN 无配对图像转换框架为基础，提出改进模型 MP-CycleGAN。针对水下图像偏色、低对比和细节模糊问题，模型引入灰世界颜色约束、结构一致性约束和感知损失约束，在无配对条件下提升增强结果的颜色自然度、结构稳定性和纹理清晰度。同时，系统集成 Gray World、CLAHE、Gamma 等传统方法作为对照，并结合 UCIQE、UIQM、PSNR、SSIM 等指标进行综合评测。

### 2. 关键技术（精简版）

本项目的关键技术包括：无配对水下图像增强技术、多约束协同优化技术、本地 Web 前端与 Python 推理服务集成技术、多方法统一接入技术、自动报告与案例归档技术，以及增强结果与 YOLO 下游检测联动验证技术。这些技术共同支撑了系统的可交互性、可展示性、可验证性和可复现性。

### 3. 技术创新（精简版）

本项目的技术创新主要体现在三个方面：一是提出面向水下场景的 MP-CycleGAN 改进模型，通过灰世界、结构和感知约束提升增强质量；二是将传统方法、原始 CycleGAN 和改进模型统一集成到一个交互式平台中，形成完整对比链条；三是把图像增强、指标评测、下游检测验证、自动报告和案例归档整合为完整软件系统，实现从“算法研究代码”到“竞赛级软件作品”的升级。
